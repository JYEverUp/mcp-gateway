系统说明书

项目定位

这是一个面向 MCP 协议的 AI 网关演示系统，目标是把大模型、工具协议和网关治理能力整合到一套 Spring Boot 工程里。

  1. 支持 MCP 会话建立、消息派发、工具列表、工具调用和资源查询。
  2. 支持工具协议映射，把外部 HTTP 接口包装成 MCP 工具。
  3. 支持 Spring AI 接入大模型，并结合工具完成问答与流式输出。
  4. 支持 API Key 鉴权、基础限流、数据库配置管理和调试页面联调。

建议演示路径

如果你要做系统讲解或现场演示，推荐按下面这条路径走，信息量足够且不会太散。

  1. 先在“连接调试”建立 SSE 会话，展示网关如何创建 Session。
  2. 再到“工具管理”查看已注册工具，说明工具如何映射到外部协议。
  3. 接着到“AI 聊天”选择数据库里的模型配置，发起流式问答。
  4. 最后回到“网关管理”展示网关详情、鉴权和管理接口。

用户使用步骤

面向普通使用者时，可以把系统理解成“先拿钥匙，再连网关，再调工具，最后让 AI 组合工具回答问题”。

  1. 输入有效的 API Key，系统会先做鉴权。
  2. 进入“连接调试”，点击“建立连接”获取 `sessionId`。
  3. 在“工具管理”确认当前网关挂载了哪些工具。
  4. 在“AI 聊天”选择模型方案，输入问题，观察流式回答和工具调用日志。

核心亮点

如果是对外介绍系统能力，这几个点最值得强调。

  1. 网关层和工具协议层拆分明确，不只是普通 CRUD 项目。
  2. 模型配置放入数据库，页面只做选择，避免把密钥硬编码在前端。
  3. AI 问答链路已切到 Spring AI，并支持流式输出。
  4. 系统已经具备 API Key 鉴权和第一版多维限流能力，具备一定治理属性。
